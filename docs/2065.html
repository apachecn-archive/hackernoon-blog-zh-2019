赋能轻网☀️👥

大多数互联网用户在网上保护自己时面临的问题是，消费者理论上可以用来保护自己的网络安全工具往往过于复杂，无法手动运行，即使它们现在已经很容易获得(下面有进一步的链接)。

用户面临的网络复杂性挑战:

  • 一个错误就可能使一个安全的过程变得不安全(即，重用相同的盐/密钥和初始化向量来加密不同的消息)。
  • 出错空间更小(即没有复位按钮)
  • 需要更加关注细节
  • 需要时间来学习新的方法

以比特为单位的预期安全假设🔬

加密原语会随着时间而改变，因为只要它们被认为是安全的，它们就会被使用，直到它们不再安全。在收敛点之前(当旧方法相对于风险不再安全时)，使用新方法来替换旧的原始方法。

加密安全性通常以二进制位来衡量，二进制位可以指密钥的长度和/或随机选择密钥的可能数字的范围(即，从2个⁸可能的128位数字的范围中选择的128位二进制数)。

观点:任何伪随机数生成器只有在生成器每次运行时，从各种输入源收集的各种熵输入才是安全的(如本文引用的W3C Crypto API中概述的)。

密码安全流程意味着应用程序所使用的底层原语和流程构造方法已经过广泛的测试，并被视为行业标准。

与此同时，新引入的提案(即Boneh等人的盲签)可能仍处于同行评审期，正在接受此类测试，尚未被采纳。

以下是显示曲线Ed-25519的BIP-39示意图的图表/流程图示例。实施这一过程来创建人类可读密钥(助记词)的加密货币钱包通常会在所得的24个字中产生256位的安全性(或12字密钥短语的128位)，不包括确定性的校验和位(哈希导出)。

破解现代密码原语🔨 💔

虽然我们已经讨论过的密码安全程序可能(很可能)在未来被破解，但就目前而言，它们被认为是足够安全的。给定可行的已知理论攻击媒介的数量，网络安全社区在合理的时间内(即，接下来的几年或直到它们预计不再安全)测试并依赖这些原语。这是与时间的赛跑。

任何这样的安全假设都意味着不可行的攻击向量是有效的风险，只是概率非常低，以至于它们被认为几乎不可能/不可能。换句话说，在给定攻击者或组织可以访问的可用技术和资源的情况下，如果要花一百万年才能猜出一个具有n位安全性的密码，那么这种风险可以忽略不计。

🔜后量子算法🚧 📐

问量子计算机可能会在几分钟或几天内破解这样的密码(参见维基百科上的多项式与非确定性多项式时间)，将风险转化为不可忽视的风险，不再提供保持密码安全所需的安全性，并需要立即进行更改。

在提交给国家标准研究所(NIST) 之后，一套新的量子抵抗算法被提议作为第二轮选择的一部分。

这些前瞻性的准备工作使包括主要标准机构在内的万维网有足够的时间在适当的时候过渡到下一代加密算法(理想情况下是在当前使用的算法被破解之前很久，并使用 Mosca定理来估计需要多长时间)。

开源首先提供了透明度🔍

开源的主要方面不在于软件是否免费，而在于底层语言和架构是否对用户可见，以供他们检查和审查完整的源代码。

将这种开放结构与基于盲目信任的闭源技术进行比较，当涉及到除了其创造者/所有者之外对所有人都隐藏的任何底层代码时，您可以看到每种方法都有利弊。

️️️™️开源许可的范围🚫

E 尽管有不同类型的开源许可(参见choosealicense.com来自 GitHub )，从更少限制和更开放，到更多限制和更少开放，最终它们在代码可见性方面都是开放的(这在本文的上下文中很重要)。

Github等网站上的开源公共库(如代码)吸引了合作，因为公众可以检查并提供纠正、改进和其他反馈，从而有助于推动开发过程。

Linux已经发展成为一个开源操作系统，它的一些分支(发行版)充满活力并被广泛使用，而其他分支由于缺乏支持/采用而逐渐消亡。下面是开源软件如何发展的一个例子，正如在Linux发行版随时间变化的树形图中所看到的:

在我看来，其他尚未被处理官方“开源”名称的开放源代码倡议(OSI) 标准机构接受的许可证仍然是开源的。对我来说，开放意味着源代码的完全可见性，即使共享和使用的次要好处受到限制，例如 MongoDB 的最新服务器端公共许可证(SSPL) 。

开源软件的网络安全风险🎯 👜

虽然开源软件可能与闭源或混合源软件一样容易受到安全漏洞的影响，但让代码完全开放可以让适当的尽职调查变得透明。

最近的一项分析发现，许多开源项目已经泄露了他们的密钥数据，这表明需要最佳实践来保护认证数据(即引用用户SSH密钥的路径，而不是引用托管文件中的实际密钥)。

Firefox最近推出了其 Firefox Send web应用，它允许用户通过使用链接向其他任何人发送加密文件，其中链接充当解密文件的密钥，并且链接可以过期，可供一个或多个收件人使用。

端到端加密，来自第三方服务🏪

使用端到端加密的服务在某些国家甚至可能是非法的(如澳大利亚，该国去年刚刚通过了新的加密法，除非实施了某些后门，这些后门会降低这种服务最初试图提供的安全性)。

其他国家正在资助端到端加密服务的开发，欧盟委员会最近向总部位于瑞士的✉️质子邮件服务提供商提供了超过100万欧元的资助，以帮助其资助bug-bounty🐛提供其开源组件的众包开发(即向帮助改进代码和发现/修复bug的开发人员付费)。

虽然开源是透明的(不像闭源代码)，但它仍然需要你自己检查代码，或者依赖维护这些库的可信第三方。

完美的安全永远不会存在🔁

就像一个洁净室只有在一个时刻才是干净的，因为一粒灰尘可能会飞进去，时间在一个被称为熵的过程中引入衰变。这是一个也在计算机科学中使用的术语(和公式)，由 Claude Shannon 从物理学中的玻尔兹曼熵方程中借用，因为它与计算随机弦的强度有相似的结构，因此熵适用于信息论。

专业提示:您可以通过计算log2(可能的组合)(即log2(passwordlibrary^passwordlength)=以位为单位的总熵来计算随机密码的熵。)

熵公式:

  • Log2(可能的组合)= 总密码熵
  • rangeofpossiblecharacters^lengthofpassword=可能的组合
  • log2(RangeofPossibleCharacters)=每个字符的熵
  • 每个字符的熵*密码长度=总密码熵

没有完美安全这种东西的原因是因为时间，自久而久之以来，渗透安全的新方法出现了，因为安全是一个过程，而不是目的地。这也是为什么当需要更多的熵时，键的大小继续增加。用许多著名的密码学家和网络安全思想领袖的话说，让我们重新思考一下这些话，“安全是一个过程，而不是目的地”。

不断发展的网络标准

T 国际标准化组织(ISO)正在制定一些相关的加密标准(即27000系列),包括区块链和网络安全标准，此外还有ISOC、ISACA、COBIT和ITU电信标准化部门(ITU-T)以及互联网安全中心(CIS)。

其他机构如万维网联盟(W3C)有一个加密API (注意:我是Github上W3C加密库的贡献者🤓).

诸如“getRandomValues ”(相当于使用Python中的“secrets”库或Linux终端上的“Dev(Random)”命令)之类的可信命令被无数的互联网应用所依赖，以获得足够安全的熵(即抗预镜像的熵),从而产生在比特(即256比特数)方面加密安全的伪随机数生成器。🙀

这些技术细节可能会让普通消费者昏昏欲睡😴 💤，但人们不需要了解哈希函数的内部工作原理(除非好奇，或者追求更高的教育)，而是需要了解它们如何与更大的网络安全图景联系起来。🖼

换句话说，许多人相信这些过程在密码上是安全的，因为如果有人可以可行地重新创建熵(前映像)，这将打破安全假设，并将用户数据置于风险之中。
