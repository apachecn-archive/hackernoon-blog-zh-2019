注意:旅游保险是一种非常普通的保险，有特定的用途。因为我们正在重新想象旅游保险是什么，而不是它是什么。从这一点上，我做出了协调一致的努力，以确保我们从正确的角度出发，考虑简单性、客户价值，并发现利用当今技术的可能性。我很早就发现很容易把事情复杂化。所以，我给自己写了一份备忘录，引用了一些我在品牌和产品早期开发时相信的话。

1.我们代表什么？

这是我所帮助的初创公司经历的最重要的步骤之一。通常，一个创始人会有一个有效的想法，概念，甚至在市场上，然而当他们开始建立/扩展它时，他们经常会错过关键的基本面；最重要的是… 我们代表什么？

看待这个问题的一个简单方法是在设计或产品流程开始之前做一些练习。其中包括:

  • 产品信念——我们必须拥有什么，我们想要避免什么。
  • 品牌叙事和形象——我们希望它变成什么样？

2.产品会是什么样子？

今天的旅游保险产品并不复杂…我们已经知道这一点。通过研究，我发现如今的旅游保险是:

  • 保险界的红发继子；
  • 主要以纸/人为中心；&
  • 构建于80年代末90年代初的大型机技术堆栈。

如果在它之前没有任何东西被创造出来，那会是什么呢？以下是一些想法:

为产品考虑:

  • 我们应该包括什么？今天人们如何旅行？在过去的十年里有什么变化吗？
  • 保险通常是用户不信任的产品。我们如何建立信任？
  • 如今的旅游保险依赖于对用户去向的了解。我们能使用地理定位技术来实现无缝连接吗？地理围栏…可以用吗？
  • 脸书全是用户出行数据..如今，每个人都在旅行时“登记”并分享照片——我们能利用这一点吗？
  • 索赔的主要原因是什么？当前的产品套件中是否缺少了什么？
  • 离开的时候天气会怎么样？现在是热带气旋季节吗？这会影响用户的旅行体验吗？
  • 了解哪些航空公司、运输公司或酒店的用户会影响我们的保险吗？如果人们使用特定的航空公司或住在特定的酒店，他们会要求更多吗？

那么，尽管如此，我们该如何解决这个问题呢？

让我们开始:

首先要移动

移动优先或本地意味着我们可以使用许多功能，包括根据旅行信息的变化或更新提供现场通知。我们可以利用智能手机的地理定位技术来了解封面是否与他们的活动相符。我们可以使用脸书SDK来收集更多关于用户的详细信息，并根据这些数据个性化体验。我们可以在网络上做类似的事情，然而，与设备上的常见行为相匹配的地理位置数据为移动设备提供了更强有力的支持。更好的是，我们中的很大一部分人在旅行时会带着智能手机，所以我们首先开发智能手机是有意义的，而不是笔记本电脑或PC。

使用REST API

如今，REST API与移动开发齐头并进。然而，就保险而言，对产品的理解和结构仍在不断发展。当然，API可以处理应用程序的基本核心/卫生功能，但是我的目标是创建连接到各种有意义的服务的微服务。如果我们接入Amadeus(旅行框架)、Smart Traveller(安全平台)和WeatherAPI，我们就会对用户、用户正在访问的地点以及旅行中会出现哪些风险有深入的了解。

添加一个. I来持续自动化

作为保险公司的一部分是收集适当的数据，根据一些商业规则进行分析，并根据风险制定价格。使用人工智能，我们可以利用最新的业务规则自动化保险的定价阶段，这些规则融合了用户的实时数据收集、位置数据和影响保单的补充数据。另一方面，我们可以使用人工智能来支持索赔流程，检查保单数据、用户数据和索赔业务规则，以自动化决策。我估计大约60个微服务可以用来验证使用人工智能的索赔。

聊天机器人能帮助自动化索赔过程吗？

这个我觉得值得在实验中锻炼。我确实认为，当意图明确且有价值时，聊天机器人是有用的。我见过的一些聊天机器人试图做太多的事情，用户经常被回答淹没或困惑。我认为对于没有太多变量的简单索赔过程，可以使用聊天机器人。对于更复杂的索赔，如医疗索赔，我们应该试图模糊聊天机器人和真实人类之间的界限，创造无缝和有价值的体验。说起来容易做起来难！

以关系为中心，而不是以产品为中心

我认为经常被忽略的一个角度是，当构建一个产品来满足当今用户的需求时，什么才是真正重要的。随着时间的推移，用户的期望已经发生了根本的变化。今天，建立关系是最重要的，所以我们希望专注于此。

3。我们卖给谁？

本质上，旅行者是最终用户。所以应该叫出来。然而，69%的旅游保险是通过“经纪人”销售的。经纪人可以是航空公司、订票网站或旅行社。在这个练习中，我将首先关注数字接口，因为我们可以使用REST API无缝集成。

对于当前的发现阶段，我首先关注旅行者，因为他们是真正的最终用户。然而，数字经纪人可能需要达到规模，因此我开始了解他们从销售旅游保险中获得的价值。简而言之，他们最多可以获得保费的50%的佣金(即如果保单成本为102美元，数字经纪人将获得51美元的佣金)。这是一笔很大的佣金…我认为在后期阶段发现它是值得的，但是这可能是颠倒的(iTunes对音乐行业？).

4。人们愿意为这种产品付费吗？什么是顾客感知价值？

这是有趣的一个。旅游保险通常是免费的附加服务，那么人们为什么要付费使用它呢？与一些保险公司谈及此事时……内部都认为是这样，因此对旅游保险开发的持续投资很低。我不同意这种心态。

有价值的时候，人们才愿意花钱。这种情况可以适用于今天的一系列免费产品或在某个时间点包含的产品。瓶装水、送餐、塑料袋、压缩空气、在线报纸，甚至日常银行业务现在都因感知价值而有所花费。

甚至Slack在一段时间内也是免费工具，直到他们创造了足够的客户价值，主要是通过他们的应用市场和达到值得花钱购买的产品抛光水平。

顾客感知价值

顾客感知价值不是一项技术，而是一个如上所述的公式。这是一个很好的工具，可以用来了解您的产品或服务是否符合客户的感知价值，以及与这些收益相关的收益和成本。

想象一下，如果你免费获得旅游保险，但是:

  • 注册需要8分钟，在最后，你仍然必须看到小字，以确保你已经涵盖了最重要的内容。
  • 不得不费力地阅读多达10份文件来完成一项索赔。
  • 必须等待长达12天才能收到钱(当前行业平均水平*)。
  • 保单上写满了旨在让你犯错的细则(请原谅这个双关语)。

万一:

  • 注册过程简单、清晰、透明且数字化，使用当今的技术来简化流程。
  • 你在旅途中带走的东西被完全掩盖了。笔记本电脑、电话、滑雪板、吉他，应有尽有。
  • 用户实际上可以详细查看他们的旅行，并被告知他们旅行中的任何风险。(在可能的情况下)甚至可以数字化登记航班和酒店。
  • 用户策略随着用户在旅程中的进展而更新。
  • 大约70%的索赔过程都是数字化的，并且对用户的旅行是个性化的。索赔应该在3分钟内自动完成，而不是12天。
  • 一旦申请被批准，资金会实时收到，而不是在12天后。
  • 如果保单在没有索赔的情况下完成，一部分保费将根据用户在财年结束时旅行的地点捐赠给选定的可持续旅游机构。

这些是我现在设定的目标，试图在建设时达到。

在这个发现过程中，我想到了一句口头禅:

我们知道，人们寻求达到一个可测量的结果，而且是可以实现的。我们希望我们的价值通过我们对理解和实现用户期望结果的承诺而被感知，而不是简单地销售一个解决方案。

5.保险公司是如何运作的？

如今，保险公司分为两类:

  1. 他们是一家成熟的保险公司，处理承保、索赔等业务。(安联、Suncorp、IAG、劳埃德都属于这一类)。
  2. 他们是“封面持有者”,这实质上意味着他们是第1点中名称的前端。(真正的保险、直接预算、Huddle属于这一类)。

第一:实现这一目标的过程漫长，成本高昂，需要大量资金。

第二:更精简，但是你需要协商产品、定价模式、索赔流程、运营模式、业务的偿付能力验证……还有很多。

通常，#2是大多数新保险公司的目标，然而，如果你希望你的产品真正以客户为中心，并符合上述原则，你需要与保险公司谈判。因此，我开始与一家保险公司进行对话，目的是共同打造一款满足当今旅行者需求的产品。通常情况下，大多数保险公司会采用现成的产品并更改品牌名称。这将是一个漫长的过程，但对于实现预期的客户价值至关重要。

6。用户目前面临的旅游保险痛点有哪些？

在为期6周的旅程中，我与32名购买了旅行保险并提出索赔的用户进行了交谈。在32个中:

  • 12人对旅游保险有过积极的体验。评论包括不需要保险，索赔过程处理迅速，在可接受的时间范围内。
  • 8人对旅游保险有一定的体验。评论包括冗长但令人满意的索赔经历。
  • 10有一次不太愉快的旅行保险经历。意见包括索赔没有得到全额支付，必须提供一系列文件，这在他们的旅行中是困难的。
  • 2名用户与他们的旅游保险提供商有过可怕的经历。

总体而言，91%的澳大利亚人在旅行时购买了旅行保险。这是一个很大的统计数据，说明旅游保险是有价值的，然而大多数人认为它是一种必要的邪恶！

真正让我震惊的是，在预订有旅游保险的航班时，一些反馈。在预订航班时，由于免费保险，用户通常希望使用他们的白金信用卡(每年150美元)来购买航班。当他们使用信用卡预订时，他们因使用信用卡而被额外收取4.0%的费用。我真的试过了，并试图预订一张去阿姆斯特丹的机票。两个人回来，一共是3788美元(没有旅游保险)。我选择了信用卡，它额外增加了4.0%或151.52美元，使总数达到3939美元。

如果我用我的借记卡预订并通过安联或Covermore (avg)购买了综合保险。150美元作为保险费)，我将得到同样数额的3939美元。这将是常见的用户行为，将需要一些教育，以改变这种已知的旅行保险路径。

注意:我要说的是，由于我希望捕捉的实时数据的绝对重要性，这些数据将推动风险模型和业务规则，我的预期是保费将比现有产品低30%……使我们的产品在定价上更具竞争力，而且数据也有力量。新的收入流可以将每个客户的收入提高到很高的终身价值。
