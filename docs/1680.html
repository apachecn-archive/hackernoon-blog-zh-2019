开源压力——谁应该获利？

原文：https://medium.com/hackernoon/open-source-tension-who-should-profit-55e529000b84#2019-03-11

Twitter 在开源上的推文一直很活跃，特别是关于当开源遇到云时你该做什么。凯尔萨斯公司的乔恩·克里斯滕森和克里斯·希克曼以及 T2 秘密公司的里奇·斯塔茨讨论了开源软件和项目的创造者和商业用户之间的紧张关系。

该展览的一些亮点包括:

  • 开源公司和赞助商被认为是理所当然的，并感到有压力去寻找赚钱的方法
  • 紧张源于过去几年经济和开源商业模式的变化；开源项目需要找出如何适应它们
  • Redislabs 修改了它的开源许可证，加入了 Commons 条款，以保护云供应商在没有贡献的情况下赚钱
  • 许可的常见商业方法是改进需要开源，否则不能从作为服务运行的软件中赚钱
  • 很难跟上许可证的步伐，因为有太多的许可证，公司现在发明自己的许可证来强制限制软件
  • 一些公司不能/不想与云竞争来赚钱和发展
  • 公司可以改变许可证来赚钱和提高竞争力，但他们不拥有开源代码/项目
  • 开源软件前提:给社区，不归任何人所有
  • 开源用户包括 AWS 的 Kafka、DocumentDB 和其他服务
  • 开源软件作为营销解决方案的两个问题:让开发者购买和使用你的软件；和大规模操作软件
  • 开源项目的原始贡献者和他们创建的公司无法与大的提供者竞争；创造新的赚钱方式和专有软件

突然间，好的产品开始上升到顶端，并被越来越多的人使用，然后，当你有了一个真正的赢家，你知道它，这就是开源的一个特点，我会说。对吗？你会这么说吗，克里斯？

克里斯:我的意思是开源是关于社区的。这是一种拥有——它建立了与软件的关系，一种真正核心、基本的关系，因此购买软件和构建软件之间有着巨大的差异。如果你开发软件，你会对它有一种拥有感，而且你会对它更加忠诚，而不仅仅是购买现成的东西。那种关系不存在。

是的，当你和像开发人员这样的人交谈时，会有很多不必要的想法，比如，“我可以把这个东西做得更好。我可以把这个东西设计得更好。”开源把所有的东西都扔出窗外，开源是好的。你能让它变得更好？提交一份 PR。

乔恩:对。没错，所以它解决了非常奇怪的营销问题，然后像 Red Hat 这样的公司，像 Confluent 这样的公司，因为开源项目的最初创始人——我实际上不太清楚我刚才提到的那两家公司的创始故事，但通常情况下，由几个创始人开始的开源项目，他们意识到他们必须抓住你的尾巴。

他们意识到他们看到他们的 GitHub 星星在上升，上升，上升，他们就像，“哦，我的天哪。我们必须对此做点什么”，然后成立一家公司。这是一个非常好的方式，让人们意识到他们有动力获得潜在的风险投资资金，并围绕开源项目建立一些东西，是的，他们大部分确实创造了这些东西，但在他们真正走得很远之前，他们把它给了世界。这是我刚才用的一个很重要的短语。他们把它给了世界。他们真的以合法的方式为世界服务。

克里斯:是的，我的意思是这就是开源软件，对吗？一旦你决定开源，这就是合同。这是它的全部前提，这是给予社区的东西，它不属于任何人；它归社区所有，这种想法是，“嘿，如果你觉得这个软件有用，就使用它，如果你能帮助它变得更好，”这一切都是在它将会发展的承诺下，它将会从那个社区变得更好。

一旦你决定把它变成一个社区项目，这就是你所做的。你现在不能试图把它变成某种你可以从中赚钱的所有者的东西。
