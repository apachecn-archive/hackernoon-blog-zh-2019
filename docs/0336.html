如何用Joget和红帽OpenShift容器平台可视化构建app

原文：https://medium.com/hackernoon/how-to-visually-build-apps-with-joget-and-red-hat-openshift-container-platform-c6e58089b866?source=collection_archive---------15-----------------------#2019-01-14

红帽OpenShift 是一个容器应用平台，为企业带来 Docker 和 Kubernetes 。作为一个云本地计算平台， OpenShift 允许团队自动化应用的构建、部署和管理。他们可以专注于开发应用，而无需担心底层基础架构，无论是在物理/虚拟服务器上，还是在公共/私有/混合云环境中。

向云原生计算的迁移是许多组织的数字化转型战略的关键部分。然而，这只是等式的一部分，因为必须解决的主要差距是在构建应用程序和自动化业务流程方面。今天，许多应用程序开发仍然利用传统的编码，因此固有的复杂性以及技术专业知识的缺乏是很大的障碍。

作为一个开源无代码/低代码平台，为编码人员和非编码人员可视化构建企业web应用， Joget 是OpenShift的理想补充，填补了空白。Joget平台现在作为一个 Red Hat认证的容器可用，所以它是可信的、安全的，并且在OpenShift上得到商业支持。本文描述了部署与 MySQL 数据库一起运行的认证Joget容器映像的步骤。

使用OpenShift Web控制台部署Joget

步骤1:创建OpenShift项目

进入 OpenShift Web控制台并登录

使用创建项目按钮创建项目，并输入所需的名称、显示名称和描述。

步骤2:部署MySQL数据库

在选中的项目下，选择添加到项目 > 浏览目录并选择 MySQL 图片。

输入适当的设置并点击创建，例如

命名空间:openshift

数据库服务名:jogetdb

MySQL连接用户名:joget

MySQL连接密码:joget

MySQL数据库名称:jwdb

步骤3:部署Joget平台

要访问 Red Hat容器目录，需要用于登录Red Hat客户门户网站的有效用户名和密码。

如果您没有帐户，可以通过注册以下选项之一来获得一个帐户:

  • 红帽开发者计划。此帐户允许您访问开发工具和程序。
  • 30天试用订阅。此帐户为您提供30天的试用订阅，您可以使用精选的Red Hat软件产品。

在所选项目下，选择添加到项目 > 部署映像并点击创建映像拉取秘密链接。在随后的弹出窗口中，键入注册表的Red Hat登录详细信息。

秘密名字:registry.connect.redhat.com

身份验证类型:映像注册表凭据

图像注册服务器地址:registry.connect.redhat.com

用户名:Red Hat帐户用户名

密码:Red Hat帐户密码

电子邮件:红帽帐户电子邮件

将机密链接到服务帐户:是

服务帐户:默认

一旦创建了秘密，选择添加到项目 > 部署图像，选择图像名称选项并键入registry.connect.redhat.com/joget/joget-v6-enterprise。点击小搜索图标从注册表加载图像细节，键入所需的名称，然后点击部署。

图像名称:registry.connect.redhat.com/joget/joget-v6-enterprise

名称:joget-V6-企业

步骤4:添加持久存储

下一步是向容器添加持久存储，用于存储配置文件和持久文件上传。在应用 > 部署下，选择应用，然后选择配置选项卡。向下滚动到卷部分，并单击添加存储链接。

在添加存储页面中，单击小的创建存储链接。使用所需的值创建新存储，例如

名称:joget-data

访问模式:共享访问(RWX)

大小:1GB(或根据需要)

回到添加存储页面，选择新创建的存储，将挂载路径设置为 /opt/joget/wflow，然后添加。

挂载路径:/opt/joget/wflow

步骤5:为外部访问创建服务路由

此时，无法从外部web浏览器访问该服务，因此您需要创建一个路由，它在一个主机名处公开一个服务。

点击应用 > 服务，选择joget-V6-企业。

选择动作 > 创建路线，将路径设置为 /jw ，点击创建。

路径:/jw

返回到概述，点击应用程序名称旁边的URL，启动Joget平台。

设置数据库

首次访问Joget平台时，将显示数据库设置页面。使用之前部署MySQL数据库时定义的值配置数据库设置，例如

数据库主机:jogetdb

数据库端口:3306

数据库名称:jwdb

数据库用户:joget

数据库密码:joget

配置成功后，Joget 应用中心将被加载。

注意:如果在数据库设置过程中遇到 504网关超时，这是由于数据库初始化花费的时间比默认的OpenShift路由超时长。实际上，您可以忽略该错误，等待几分钟，然后再次从Overview页面访问应用程序URL。

您还可以使用OpenShift CLI增加路由超时，即

oc注释路线$ APP _ NAME覆盖haproxy.router.openshift.io/timeout=60s

30分钟搭建一个完整的App，不用代码！

随着Joget平台的准备就绪，现在可以在30分钟内构建一个具有仪表盘、日历、审批工作流和数据管理的成熟应用程序。此外，它可以在没有编码的情况下完成，您实际上只需拖放、点击和配置即可。也可以使用一个触摸设备来构建它。
